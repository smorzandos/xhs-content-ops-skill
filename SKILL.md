# 小红书内容运营优化 Skill

这个 Skill 负责小红书内容工作里的运营包装层。

它把三类工作合并在一个流程里：

1. 导出样本的 benchmark 分析
2. 发布前的标题、关键词、封面、首屏优化
3. 平台分发逻辑下的形式判断与风险修正

适用场景：

- 用户给了小红书导出的 CSV 或分析报告，想判断什么内容在跑
- 用户已经有选题、提纲或半成稿，想优化标题、封面、首屏与关键词
- 用户想判断一条内容更适合视频、图文还是双发
- 用户想知道某个题为什么适合小红书，或者为什么不适合

不要用这个 Skill 来从零写完整正文。它更适合用于选题已定之后、草稿成型之后、发布前最后一轮优化。

## 核心输出

根据用户请求，输出以下部分中的一部分或全部：

- 选题与平台匹配度判断
- benchmark 规律总结
- 标题备选方案
- 搜索词与关键词埋点建议
- 封面文案建议
- 首屏钩子建议
- 发布形式建议：视频 / 图文 / 双发
- 风险提示：太泛、太技术、太空、太像噱头、太难分发

## 工作流程

### 1. 先判断任务类型

优先归类当前请求：

- `数据分析`：用户给了 CSV、截图、样本报告，或让你总结赛道规律
- `草稿优化`：用户给了选题、提纲、半成稿
- `发布包装`：用户接近定稿，需要标题、封面、首屏、SEO
- `规则咨询`：用户问平台分发逻辑、发布形式、包装原则

### 2. 按需读取 reference

不要默认把所有 reference 都读一遍，按任务选择：

- 平台与分发逻辑：
  - `references/platform-rules.md`
- 样本中已归纳出的有效模式：
  - `references/benchmark-insights.md`
- 标题与包装公式：
  - `references/title-formulas.md`
- 关键词与搜索词埋点：
  - `references/keyword-clusters.md`

### 3. 先对齐用户目标，再做优化

在产出建议前，先判断：

- 目标受众是谁
- 内容类型是什么：教程 / 案例 / 观点 / 关系解释
- 这条内容主要想拿什么：收藏 / 搜索流量 / 评论互动 / 新粉关注

不要脱离目标，机械地产出“更好的标题”。

### 4. 尽量给具体产出，而不是只讲道理

如果用户给的是成熟选题或草稿，优先直接给：

- 5-10 个标题
- 2-3 套关键词埋点方向
- 2-3 条封面文案
- 1 个推荐首屏钩子
- 1 个发布形式建议

而不只是抽象点评。

### 5. 使用外部数据时要克制

当用户给了 benchmark 数据时：

- 归纳样本中的重复模式
- 尽量区分高收藏和高评论内容
- 区分真实需求与 curiosity bait
- 不要机械模仿高噱头表达

如果结论只是来自当前样本，而不是平台官方规则，要明确说这是基于样本的推断。

### 6. 站内调研前先确认是否真的可做

如果用户要求直接做站内检索或自动化调研：

- 先确认浏览器自动化是否可用
- 再确认是否需要登录态
- 如果出现安全限制、风控页、461 等异常，不要假装已经完成站内调研

调研失败时，优先改用：

- 用户导出的 CSV
- 用户提供的截图
- 公开网页与官方资料
- 已沉淀的 benchmark 结论

## 优化原则

### 标题

好的小红书标题，通常至少做到以下一项：

- 降低门槛：`小白/普通人/文科生也能`
- 压缩复杂度：`一个视频搞懂`、`到底是什么关系`
- 承诺具体结果：`用它做了…`、`打造了…`
- 制造现在就该点开的理由：`先学什么`、`值不值得`、`没必要`

避免：

- 没有受众、没有结果的空标题
- 纯术语堆砌
- 只有情绪，没有实际承诺

### 搜索词埋点

关键词应自然嵌入以下位置：

- 标题
- 副标题或开头第一句
- 小标题
- 封面文案

默认使用：

- 1 个主关键词簇
- 2-4 个辅助关键词

不要堆满词。

### 发布形式选择

默认判断规则：

- `视频`：安装、演示、上手过程、操作 walkthrough
- `图文`：概念解释、关系图谱、结构化清单、判断力内容
- `双发`：既有搜索需求又有叙事性的高价值教程或案例

### 信任优先

这个 Skill 必须优先保护创作者长期信任，而不是短期点击。

优先：

- 具体
- 真实场景
- 清晰受众
- 有边界感

避免推动用户去做：

- 正文撑不住的 clickbait
- 过度许诺
- 空泛的“AI 改变人生”式表述

## 输出模板

### 如果是数据分析

输出结构：

1. 当前样本里什么在跑
2. 这些内容满足了用户什么需求
3. 哪些表达值得借
4. 哪些不能照抄
5. 对用户自己的选题与包装有什么影响

### 如果是草稿优化

输出结构：

1. 目标受众
2. 推荐定位
3. 标题备选
4. 关键词簇建议
5. 首屏钩子
6. 封面文案
7. 发布形式建议

### 如果是发布包装

输出一份短版发布包：

- 主标题
- 备选标题
- 封面文案
- 首屏文案
- 搜索词埋点建议
- 发布形式建议
