# 小红书 AI 赛道爆款分析报告

*基于 `/00inbox/draft/xhs数据导出-0320/` 内 2026-03-20 导出的多组关键词样本整理*

## 1. 分析范围与方法

这次不是泛泛看“小红书上 AI 很火”，而是重点分析和你接下来要做的内容最接近的赛道：

- `cc`
- `agent`
- `openclaw低粉爆文`
- `vibecoding`
- `文科生学AI`
- `skill`
- `mcp`
- `cursor`
- `AI入门`

处理原则：

1. 先按笔记链接去重，避免同一篇在多个关键词下重复计算。
2. 优先看和你方向最接近的内容，不让泛 AI 噪音误导判断。
3. 从三个层面分析：
   - 选题为什么被点开
   - 标题和表达为什么容易传播
   - 从创作者和受众各自视角，内容满足了什么需求

## 2. 总体判断

你的赛道在小红书上已经被验证过，且不是只能靠大号跑通。

最重要的结论有 5 条：

1. **安装教程真的有需求，而且是高收藏需求。**
2. **“文科生/小白/普通人也能上手”是强钩子。**
3. **单纯教工具不够，最好是“工具 + 场景结果 + 身份代入”。**
4. **关系解释型内容比纯工具罗列更容易形成高收藏。**
5. **反常识观点和真实踩坑，最容易带来评论和讨论。**

也就是说，这个赛道不是没人做，而是大家已经证明了：

- 用户确实想学
- 用户并不想听太技术的讲法
- 用户最爱的是“看起来离自己不远，但比自己先进半步”的内容

## 3. 哪些内容最容易火

### 3.1 安装部署教程

这是目前最稳定的一类。

代表标题：

- 手把手保姆级配置本地 OpenClaw 教程
- 人人都能用的 Claude Code 安装指南
- OpenClaw 保姆级安装教程
- 从 0 开始安装｜小白教程

数据表现：

- 在焦点样本中，`安装部署教程` 类平均互动约 `11179`
- 平均收藏约 `6050`

为什么火：

1. 用户的真实卡点就在第一步，安装焦虑很强。
2. 装成功意味着“我也进入这个世界了”，成就感明确。
3. 这类内容天然适合收藏，用户会想“以后要用”。
4. “保姆级 / 手把手 / 从0开始”直接降低恐惧。

创作者视角：

- 这是最适合用来吸新粉的内容。
- 但同质化已经开始明显，不能只停在“步骤罗列”。
- 真正更强的做法是：安装教程里顺带讲“为什么要装、适合谁、不适合谁”。

受众视角：

- 他们收藏的不是知识，而是“未来可能会动手的机会”。
- 用户看这类内容时的核心心理不是学习，而是减轻门槛感。

### 3.2 身份代入型内容

这类标题很适合你。

代表标题：

- 文科生 5 分钟拥有自己的 AI 助理
- 不写代码的我用 Claude Code 当职场替身
- 不懂代码的我，第一次用 Claude Code
- 开工第一天，一个文科生自己跑通了 OpenClaw

为什么火：

1. 它不是在讲工具，而是在讲“像我这样的人能不能学会”。
2. “文科生”“不懂代码”“普通人”“人人都能”都在替用户说心里话。
3. 这类内容比纯教程更有代入感，门槛被人格化地拆掉了。

创作者视角：

- 这是你最该抢的位置。
- 你不是最强技术流，但你更适合讲“普通人怎样一步步升级”。
- 这个赛道里，强势身份不是“资深工程师”，而是“已经替你踩过坑的人”。

受众视角：

- 他们想找的不是老师，而是“和我差不多，但已经走过去的人”。
- 他们买单的是心理安全感，不只是信息。

### 3.3 玩法/案例/结果导向内容

代表标题：

- Openclaw 3 大超实用玩法，炒股，搞情报...
- 用 Claude Code 打造我的论文流水线
- 看我让 3 只龙虾，在飞书群里“开日会”
- Openclaw！我养了一只会做咨询级 PPT 的龙虾

为什么火：

1. 用户对“工具本身”不一定有兴趣，但对“它能替我干什么”有兴趣。
2. “论文流水线”“开日会”“做 PPT”“搞情报”都把抽象技术翻译成了具体结果。
3. 这类内容通常评论更多，因为用户会立刻联想到自己的场景。

创作者视角：

- 这类内容比安装教程更能建立你的独特性。
- 因为案例来自你的真实生活，别人难以复制。
- 也是最适合后续接系列的内容：从“装了”到“怎么用”。

受众视角：

- 用户不是来学概念的，是来确认“这东西对我到底值不值得”。
- 他们希望看到可迁移的用途，而不是看技术炫技。

### 3.4 关系解释型内容

代表标题：

- OpenClaw、大模型、Token 到底是什么关系
- LLM、Agent、skills 是什么关系
- 22s 搞清！8 大场景 AI 如何选择【专业/平替】
- 一个视频搞懂 OpenClaw

数据表现：

- 焦点样本中，`关系科普/解释` 类平均互动约 `15508`
- 平均收藏约 `7953`

这是一个非常强的信号：**解释型内容虽然看起来不炸，但非常适合积累高价值用户。**

为什么火：

1. AI 名词太多，用户的真实痛点是认知混乱。
2. “是什么关系”“怎么选”“一篇搞懂”能直接承接混乱感。
3. 这类内容满足的是“整理世界”的需求，所以收藏率高。

创作者视角：

- 这是你最值得长期占领的内容类型。
- 因为你本来就擅长做“工具翻译”。
- 如果你只做安装教程，会变成同质化工具号；但如果你持续做“关系解释”，你会更像一个可信的向导。

受众视角：

- 用户并不缺工具信息，缺的是清晰地图。
- 一旦有人把关系讲明白，就容易被当作“值得关注，后面还会继续讲清楚的人”。

### 3.5 反常识/观点型内容

代表标题：

- 深度玩了一个多月 OpenClaw，普通人没必要装
- 提示词的时代快结束了，下一个是什么？
- 一个让安装 OpenClaw 卖家集体失业的网站
- 文案工作者不会被 AI 取代的证据

数据表现：

- `反常识/观点` 类平均互动约 `15554`
- 平均评论显著高于教程类

为什么火：

1. 它直接打破用户已有预期，容易让人停下来。
2. 观点型内容更容易引发站队、质疑、认同，因此评论更高。
3. 在 AI 赛道，用户对“追不追风口”很焦虑，反常识观点会激发强讨论。

创作者视角：

- 这是建立人格和判断力的关键。
- 但前提是你要真有经验，不能为了反常识而反常识。
- 最好的形式是：先展示你真的玩过，再说为什么不建议所有人都跟。

受众视角：

- 用户其实很怕“又被教育一个新玩具必须学”。
- 所以“没必要装”“别急着跟风”这种内容，会让人觉得你更可信。

## 4. 标题为什么有效

从标题层面，这批数据有非常清楚的规律。

## 4.1 最常见的高效标题元素

### 元素 A：降门槛

高频词：

- 保姆级
- 手把手
- 小白
- 从0开始
- 人人都能
- 不懂代码
- 文科生
- 普通人

作用：

- 降低陌生感
- 把用户从“我不会”拉到“也许我可以”

本质上，这是一种 **心理许可**。

### 元素 B：压缩认知负担

高频表达：

- 一个视频搞懂
- 22s 搞清
- 一篇看懂
- 到底是什么关系
- 怎么选

作用：

- 把复杂问题包装成可被快速吸收的答案
- 符合小红书用户“先快速判断值不值得深看”的浏览习惯

### 元素 C：承诺具体结果

高效表达：

- 拥有自己的 AI 助理
- 打造论文流水线
- 在飞书群里开日会
- 做咨询级 PPT

作用：

- 让工具不再抽象
- 让用户能想象自己使用后的状态

### 元素 D：带一点刺激性

常见表达：

- 震撼
- 没必要
- 失业
- 下一个是什么
- 集体失业

作用：

- 增加停留和评论
- 制造争议感和信息差

### 元素 E：数字与时间

常见写法：

- 3 大玩法
- 5 分钟
- 22s
- 一个月
- 一周 AI 大事

作用：

- 给用户明确预期
- 让信息显得更“可消费”

## 4.2 爆款标题不是在卖知识，而是在卖一种低风险升级

这批标题最厉害的地方，是它们都隐含了一个承诺：

**你不用先变成工程师，也可以先获得一点 AI 能力。**

这就是为什么“文科生”“小白”“人人都能”“一个视频搞懂”特别强。

它们卖的不是课程，而是：

- 我帮你减轻焦虑
- 我帮你先走一步
- 我帮你看懂这个世界

## 5. 什么样的内容更适合收藏，什么样的内容更适合评论

### 5.1 收藏型内容

更偏向：

- 教程
- 工具关系解释
- 安装步骤
- 资源/skill 合集
- “怎么选”型内容

用户收藏的原因通常是：

1. 以后可能用得到
2. 现在没空细看
3. 这个信息结构化，值得回看

对你来说，收藏型内容适合作为：

- 系列前几篇
- 搜索入口内容
- 新粉引流内容

### 5.2 评论型内容

更偏向：

- 反常识观点
- 有争议的判断
- 强代入身份
- 真实踩坑和立场表达

用户评论的原因通常是：

1. 他有意见
2. 他感到被说中了
3. 他要补充自己的经验
4. 他想问你下一步怎么做

对你来说，评论型内容适合作为：

- 建立人格
- 强化“你不是搬运号”
- 测试观点边界

## 6. 从创作者角度看：这些内容为什么能跑出来

### 6.1 这类内容的核心不是技术，而是翻译能力

跑出来的创作者，大多不是靠技术深度本身赢，而是靠：

- 讲得不吓人
- 讲得像自己真的用过
- 讲得让普通人觉得“我也能试”

也就是说，**平台奖赏的不是最懂的人，而是最会转译的人。**

这点非常适合你。

### 6.2 平台上“半步领先”的人比“顶级高手”更容易火

因为用户想看的不是大神秀操作，而是：

- 和我差不多背景的人
- 已经先跑通了一步
- 能告诉我最小可行路径

这也是为什么低粉账号能冲出来：

- `用 Claude Code 打造我的论文流水线`
- `文科生5分钟拥有自己的AI助理`
- `一个视频搞懂 OpenClaw`

这些都不是“我世界第一懂”，而是“我替你先走了一步”。

### 6.3 爆款内容常常同时满足“搜索流量 + 推荐流量”

比如：

- `Claude Code 安装指南`
- `OpenClaw 教程`
- `文科生 AI 助理`

它们既能被主动搜到，也能在推荐流里靠标题和身份钩子被点开。

这对你尤其重要，因为你不是纯娱乐博主，还是要吃搜索流量的。

## 7. 从受众角度看：他们真正想要的不是教程本身

用户真正想解决的，通常不是“怎么安装”。

他们真正想解决的是：

1. **我是不是已经落后了**
2. **我这种人能不能学会**
3. **我学这个值不值得**
4. **我学会以后能拿它干嘛**
5. **我会不会折腾半天最后白费**

所以真正有效的内容，要同时回答两层问题：

- 表层：这个工具怎么装、怎么用
- 深层：它为什么值得你现在学，什么人适合，什么人不适合

这就是为什么纯参数介绍不容易形成强连接，而“普通人没必要装”反而很容易被讨论。

## 8. 对你的启发：该学什么，不该学什么

## 8.1 你最该学的

### 学“讲解顺序”

不是一上来教安装，而是：

1. 先解释这东西是干什么的
2. 再解释什么人适合
3. 再给一个真实场景
4. 最后再讲怎么上手

### 学“身份翻译”

把内容翻译成：

- 文科生也能懂
- 普通职场人也能上手
- 不会写代码也可以先做这三件事

### 学“具体用途”

永远不要只说工具名，要说：

- 它能替你做什么
- 它在你生活里是怎么出现的
- 你为什么会继续用它

### 学“保留判断力”

多做这类内容：

- 什么人适合装，什么人不适合
- 什么情况下别折腾
- 你先学这个，不要先学那个

这会比纯教程更建立信任。

## 8.2 你不该学的

### 不要学纯噱头

比如只追：

- 程序员要失业了
- 一句话生成 App
- 龙头工具又杀疯了

这类内容虽然能起量，但会稀释你的长期定位。

### 不要学过度许诺

比如：

- 5分钟彻底学会
- 人人都能轻松赚到
- 零基础无脑部署就能改变人生

这种表达短期有效，但长期伤信任。

### 不要学“只有步骤，没有立场”

这会让你变成一个很容易被替代的教程号。

你真正该做的是：

- 步骤给到
- 但顺便告诉用户：这件事值不值得做

## 9. 对你最有价值的内容组合建议

你最适合的，不是一味追“安装教程”，而是做一个三层组合。

### 第一层：吸新粉

做这些：

- 安装指南
- 一个视频搞懂
- 文科生也能上手
- 普通人先学什么

作用：

- 降门槛
- 拿搜索流量
- 拿收藏

### 第二层：建立信任

做这些：

- 我第一次用 Claude Code 做了什么
- OpenClaw 适合什么人，不适合什么人
- 我为什么继续用 / 为什么放弃某种玩法

作用：

- 从教程号变成“可信的实践者”

### 第三层：建立差异化

做这些：

- Cloud Code、OpenClaw、Agent、IDE 到底是什么关系
- 什么是 AI Native 工作流
- AI 时代的人机边界
- 真正拉开差距的不是会不会装，而是会不会管理 AI

作用：

- 这部分才是你长期最不容易被替代的资产

## 10. 可直接给你的结论

如果只讲一句话：

**这个赛道能火，不是因为大家爱学技术，而是因为大家想低风险地完成一次身份升级。**

他们想从：

- 不懂 AI
- 害怕技术
- 只会聊天

升级成：

- 能看懂工具关系
- 能装起第一个 Agent
- 能让 AI 帮自己做一点真实的事

而你很适合做这个升级路径的讲解者。

## 11. 下一步建议

基于这份分析，你接下来最值得做的不是继续抓更多关键词，而是：

1. 从你本地知识库里筛出最适合做成“普通人 AI Native 入门系列”的题。
2. 先定第一批 8-10 个题，分成：
   - 认知破冰
   - 工具关系
   - 最小上手
   - 判断力
3. 后续每个题都按这套结构写：
   - 这个东西是什么
   - 什么人适合
   - 它能干什么
   - 我真实怎么用
   - 你第一步怎么开始
